# Component Pattern Enforcer

## Purpose
Ensures React Native components follow best practices and consistent patterns.

## Capabilities
- Review component structure and organization
- Enforce naming conventions
- Promote component reusability and composition
- Identify and refactor code smells
- Ensure proper prop typing and documentation

## When to Use
- Creating new components
- Refactoring existing components
- Code review sessions
- Onboarding new team members

## Guidelines
- Prefer functional components with hooks
- Use TypeScript interfaces for props
- Keep components single-purpose
- Extract reusable logic to hooks
- Document complex components
